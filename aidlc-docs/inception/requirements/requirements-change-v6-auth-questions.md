# v6 변경요청 확인 질문 — 회원가입/로그인 + 회원별 데이터 관리

**요청 요약**: 회원가입/로그인 화면을 만들어 회원별로 데이터를 관리하고, 지금까지 입력한 데이터(가구 프로필 등)는 본인 계정으로 이전한다.

**영향 범위(중요)**: 현재 앱은 "단일 가구 전용" 전제(C-7)로 설계되어 있습니다 — 프로필 1건, 추천도 그 프로필 기준 하나, 푸시도 전체 구독자 발송. 회원별 관리로 바꾸면 DB 스키마(user_id + RLS), 추천 재계산(회원별), 푸시(회원별 대상), 프로필 API, 공고분석까지 걸쳐 수정됩니다. v2 이후 가장 큰 변경입니다.

**보안 안내**: 비밀번호는 저(AI)에게 알려줄 필요가 없습니다. 가입 화면에서 직접 입력하고 Supabase Auth가 해시로 저장합니다.
