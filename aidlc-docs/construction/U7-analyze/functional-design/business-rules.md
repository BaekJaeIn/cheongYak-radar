# U7 공고분석 — 비즈니스 규칙 (v4, FR-12)

- **BR-U7-1 (입력 검증)**: `application/pdf`만 허용, 파일 크기 3MB 초과 시 업로드 거부 + 크기 초과 안내 (C-10). 검증은 클라이언트(즉시 피드백)와 서버(방어) 모두 수행.
- **BR-U7-2 (구조화 추출)**: Gemini 추출은 JSON 강제(responseMimeType). 원문에 없는 값은 null — 추측 금지 프롬프트 명시. `isNotice=false`(공고로 인식 불가) 또는 JSON 파싱 실패 시 "공고 PDF로 인식할 수 없음" 안내 후 종료.
- **BR-U7-3 (프로필 비전송)**: 가구 프로필은 Gemini에 전송하지 않는다. LLM에는 PDF만 전달 (FR-12.3, Q2=A).
- **BR-U7-4 (판정 재사용)**: 판정은 기존 `EligibilityMatcher.evaluate()` + `loadCriteriaTable()`(연도 config)로만 수행 — U7에서 판정 로직을 새로 만들지 않는다 (FR-12.4). scorer/rank는 미사용(단건 판정).
- **BR-U7-5 (검증 가능 결과)**: 결과 화면에 공급유형별 판정(가능/조건부/불가 + 사유)과 함께 **LLM이 추출한 기준을 병기**하고, 참고용 고지 문구를 항상 표시 (FR-12.5, C-11).
- **BR-U7-6 (미저장)**: PDF·추출 결과·판정 결과를 저장하지 않는다. 요청 처리 후 폐기 (FR-12.6, Q3=A).
- **BR-U7-7 (프로필 부재)**: 가구 프로필 미등록 시 판정 불가 — "내 프로필에서 가구 정보를 먼저 입력하세요" 안내 + 프로필 탭 링크.
- **BR-U7-8 (오류 처리)**: Gemini 오류/타임아웃·Edge 호출 실패는 비차단 사용자 메시지("분석 실패 — 잠시 후 다시 시도")로 변환. 서버 로그에만 상세 기록. 실패가 다른 기능에 영향 없음 (NFR-4 정신).
- **BR-U7-9 (키 보호)**: GEMINI_API_KEY·SUPABASE_SERVICE_ROLE_KEY는 서버(Edge/Vercel 서버 런타임)에만 존재 — 클라이언트 번들 노출 금지 (NFR-3).
