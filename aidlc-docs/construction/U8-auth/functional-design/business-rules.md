# U8 인증·회원별 데이터 — 비즈니스 규칙 (v6)

- **BR-U8-1 (전체 잠금)**: 미로그인 요청은 `/login`으로 리다이렉트 (FR-13.3). 예외 — `/login`, 인증 콜백, PWA 자산(manifest.json·sw.js·아이콘·favicon): 이들이 잠기면 홈화면 설치·푸시가 깨짐.
- **BR-U8-2 (가입 공개)**: 이메일+비밀번호로 누구나 가입(Q2=B, C-13). 이메일 확인은 끔(D-9) — 가입 즉시 로그인.
- **BR-U8-3 (비밀번호)**: 앱 코드·DB·로그에 평문 비밀번호를 저장·전송·기록하지 않는다. Supabase Auth SDK 직접 호출만.
- **BR-U8-4 (본인 데이터만)**: household_profile·recommendations·bookmarks·push_subscriptions는 RLS로 `user_id = auth.uid()` 행만 접근. anon 정책은 전환 완료 후 제거(D-3).
- **BR-U8-5 (프로필 회원화)**: 프로필 저장·조회는 로그인 회원의 user_id 스코프. 프로필 변경 시 **본인 추천만** 재계산 (D-5).
- **BR-U8-6 (회원별 추천)**: 수집(cron) 재계산은 프로필 보유 회원 전원을 각자 프로필로 평가. 회원 간 격리 — 한 회원 실패가 다른 회원 계산을 막지 않는다(기존 공고 격리 패턴 확장).
- **BR-U8-7 (회원별 푸시)**: "신규 추천" 알림은 해당 추천이 생긴 회원의 구독 기기에만 발송 (FR-14.3). 구독 저장 시 로그인 회원에 귀속.
- **BR-U8-8 (북마크)**: 북마크는 DB(user_id+notice_id)가 원본. 첫 진입 시 기기 localStorage 잔존 목록을 1회 병합(upsert, 충돌 무시) 후 로컬 키 삭제 (FR-14.4).
- **BR-U8-9 (공고분석 회원화)**: /api/analyze는 로그인 회원 확인 후 그 회원의 프로필로 판정 (FR-14.5). 프로필 없으면 기존 profileMissing 흐름.
- **BR-U8-10 (귀속)**: `jiback96@naver.com` 가입 시 user_id가 비어 있는 기존 프로필·푸시 구독을 그 계정으로 귀속(FR-15). 귀속은 1회성(user_id null 조건)이며 다른 계정에는 영향 없음.
- **BR-U8-11 (API 인증)**: /api/profile·/api/subscribe·/api/analyze는 세션 없으면 401. service_role 사용 경로는 세션 확인 후 명시적 user_id 스코프로만.
- **BR-U8-12 (전환 안전)**: anon 읽기 정책 제거(0013)는 신규 코드 배포 확인 후 적용 — 구버전 클라이언트가 빈 화면이 되는 시간을 최소화.
